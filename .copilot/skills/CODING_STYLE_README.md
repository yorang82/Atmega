# README 작성법 가이드

프로젝트 루트의 README.md 파일은 다음 형식을 따릅니다:

## 개발 환경

--------------------------------------------------------------------------------
1. 파일명1 : 설명
중요 정보 섹션 (엔지니어링 모드, 설정 방법 등)
1. 단계별 설명
2. 추가 정보
================================================================================

----------------------
[YYYY-MM-DD] vX.XX
----------------------
1. 주요 변경 사항
   - 상세 설명
   - 세부 항목

2. 다음 변경 사항
   - 상세 설명

### 버전 히스토리 작성 규칙

1. **날짜와 버전**: `[YYYY-MM-DD] vX.XX` 형식
2. **번호 매기기**: 각 주요 변경 사항은 숫자로 구분
3. **들여쓰기**: 세부 항목은 `-` 로 시작하고 3칸 들여쓰기
4. **명확한 설명**: 무엇을, 왜, 어떻게 변경했는지 기록

### 예시
```markdown
----------------------
[2025-12-17] v0.01
----------------------
1. ADC 드라이버 HAL → LL 라이브러리로 변경
   - adc_driver.c, adc_driver.h 수정
   - LL API 사용으로 성능 향상 및 메모리 최적화
   - 채널 선택, 변환 시작, 데이터 읽기 모두 LL 함수로 변경

2. 코딩 스타일 가이드 생성 (CODING_STYLE.md)
   - 파일 헤더 형식 정의 (Author: Young Kwan CHO, Lucy)
   - 섹션 구분 방식 통일
   - Doxygen 스타일 함수 주석
   - 변수/Define 인라인 주석 규칙
```

### 작성 시 주의사항

- 최신 버전이 위에 오도록 역순 배치
- 각 버전 구분선은 `------` 22개 사용
- 섹션 구분선은 `====` 80개 사용
- 하위 구분선은 `----` 80개 사용
- 변경 사항은 기술적 내용 중심으로 작성
- 파일명이나 함수명은 정확하게 명시


## README/변경이력 작성 규칙 안내 (2026-01-05 추가)

* README.txt 변경이력(업데이트 내역) 작성 시, 상위 1~14번(기본/고정 영역)은 절대 수정하지 말 것
* 20번 줄(15번째 줄) 이후부터 자유롭게 변경이력, 버전 기록, 주요 변경점 등을 추가/수정할 것
* 예시)
